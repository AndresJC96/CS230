# CS 230 Portfolio Reflection

## Project Summary

The client for this project was The Gaming Room, a company that wanted to expand its game, *Draw It or Lose It*, from Android to support multiple operating systems through a web-based application. The goal was to design a scalable software solution that would allow multiple teams and players to connect while ensuring only one instance of the game service existed at a time.

## What I Did Well

I think I did a good job evaluating different operating platforms and explaining why certain technologies would better meet the client's needs. I also clearly documented the software requirements and provided recommendations that considered scalability, security, performance, and maintainability.

## Helpful Parts of the Design Process

Working through the software design document before writing code helped me better understand the project requirements and think through potential challenges ahead of time. It made the coding process easier because I already had a plan for how the application should be structured.

## What I Would Improve

If I were to revise one part of this project, I would spend more time expanding the security and distributed systems sections. I have learned more throughout the course, and I think I could provide more detailed recommendations for protecting the application and improving reliability.

## Understanding User Needs

I interpreted the user's needs by focusing on creating a system that was reliable, scalable, and accessible across multiple platforms. Keeping the user's needs in mind is important because the software should solve the client's problem while providing a good experience for the people who will actually use it.

## My Software Design Approach

I approached the design by first understanding the requirements, comparing different technologies, and then selecting the options that best matched the client's goals. In future projects, I will continue using this approach by breaking complex problems into smaller pieces, evaluating multiple solutions, and considering factors such as performance, security, scalability, and long-term maintenance before writing code.
